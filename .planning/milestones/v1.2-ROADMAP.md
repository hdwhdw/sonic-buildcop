## Key Decisions

| Decision | Rationale |
|----------|-----------|
| New `enrichment.py` module | Follows staleness.py pattern — separate enrichment modules for each data concern |
| Batch PR search (1-2 API calls) | GitHub Search API has 30 req/min limit; batch avoids per-submodule queries |
| Longest-name-first matching | Prevents `sonic-swss` from stealing `sonic-swss-common` matches |
| Check Runs API primary | Modern CI reports via Check Runs, not legacy Commit Statuses |
| Toggle icon in dedicated column | Avoids click-target conflict with existing links in table rows |
| Vanilla JS (~15 lines) | No framework needed for simple DOM toggle |
| Vertical key-value list | Simplest layout, stacks well on mobile |
| Muted placeholder for nulls | "No open PR", "Not enough data" — informative without being noisy |
