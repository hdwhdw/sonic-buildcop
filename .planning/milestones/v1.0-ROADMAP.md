# Milestone v1.0: Submodule Staleness Dashboard

**Status:** ✅ SHIPPED 2026-03-21
**Phases:** 1-3
**Total Plans:** 7

## Overview

Built a GitHub Pages dashboard that tracks submodule staleness in sonic-net/sonic-buildimage. The dashboard uses cadence-aware thresholds derived from each submodule's own development pace to classify staleness as green/yellow/red, and deploys automatically via a daily GitHub Actions cron.

## Key Accomplishments

1. **Data collection pipeline** — Python collector fetches .gitmodules, resolves branches, computes commits-behind and days-behind for 10 sonic-net submodules via GitHub API (~34 calls/run)
2. **Cadence-aware staleness model** — Median inter-commit interval per submodule, with 2×/4× multipliers for yellow/red thresholds; worst-of days vs commits determines final classification
3. **Polished dashboard** — Sorted worst-first, emoji summary line, responsive layout, green/yellow/red badges, deployed to GitHub Pages
4. **Automated CI/CD** — Daily cron at 6 AM UTC + manual dispatch, zero-config GITHUB_TOKEN auth, artifact-based Pages deployment
5. **57 unit tests** — Full coverage across collector, staleness, and renderer modules

## Milestone Summary

**Key Decisions:**
- Python for all scripts (SONiC ecosystem consistency, pre-installed on runners)
- Separate modules: collector.py → staleness.py → renderer.py with JSON interchange
- Cadence from submodule's own commit history (development velocity, not pointer update frequency)
- Median not mean for cadence computation (resists holiday/burst outliers)
- Both days-behind and commits-behind evaluated, worst wins
- 2×/4× fixed multipliers, fallback 30d/10c for sparse repos
- Pre-sorted in Python (no JavaScript), horizontal scroll for narrow screens

**Issues Resolved:**
- `str.rstrip('.git')` mangles `sonic-gnmi` → used `removesuffix('.git')` instead
- `requests` not pre-installed on GitHub Actions runners → added to requirements.txt
- `collect_submodule` missing `owner`/`repo` fields → staleness enrichment silently failed

**Technical Debt:**
- None significant — clean v1 delivery
