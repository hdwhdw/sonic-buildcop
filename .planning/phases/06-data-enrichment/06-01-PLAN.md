**Functions to implement (in this order):**

1. `match_pr_to_submodule(pr: dict, submodule_names: list[str]) -> str | None`
   - Takes a PR dict (from Search API) and a list of submodule names ALREADY sorted longest-first
   - Scans `pr["title"].lower()` for each name (lowered)
   - Returns first match, or None
   - Critical: caller must sort names longest-first to avoid "sonic-swss" matching before "sonic-swss-common"

2. `get_ci_status_for_pr(session: requests.Session, pr_number: int) -> str | None`
   - GET `/repos/{PARENT_OWNER}/{PARENT_REPO}/pulls/{pr_number}` to get `head.sha`
   - GET `/repos/{PARENT_OWNER}/{PARENT_REPO}/commits/{head_sha}/check-runs` with header `Accept: application/vnd.github+json`
   - If no check_runs: return None
   - If any run has `status != "completed"`: has_pending = True
   - If any completed run has `conclusion not in ("success", "neutral", "skipped")`: has_failure = True
   - Return "fail" if has_failure, "pending" if has_pending, else "pass"
   - Wrap entire function in try/except (requests.RequestException, KeyError) → return None

3. `fetch_open_bot_prs(session: requests.Session, submodules: list[dict]) -> None`
   - Initialize ALL submodules' `open_bot_pr` to None first
   - Build query: `repo:{PARENT_OWNER}/{PARENT_REPO} author:{BOT_AUTHOR} is:pr is:open`
   - GET `/search/issues` with params `{"q": query, "per_page": 50, "sort": "updated", "order": "desc"}`
   - On API error: return (all already None)
   - Build `sub_by_name` dict mapping name→submodule for status="ok" submodules
   - Sort names longest-first: `sorted_names = sorted(sub_by_name.keys(), key=len, reverse=True)`
   - For each PR in response items:
     - Call `match_pr_to_submodule(pr, sorted_names)`
     - If match found:
       - Compute `age_days = round((now - created_at).total_seconds() / 86400, 1)` where `created_at` is parsed from `pr["created_at"]`
       - Call `get_ci_status_for_pr(session, pr["number"])`
       - Set `sub_by_name[matched_name]["open_bot_pr"] = {"url": pr["html_url"], "age_days": age_days, "ci_status": ci_status}`
       - Remove matched name from sorted_names to prevent double-matching
       - `time.sleep(0.5)` after CI status call

4. `fetch_merged_bot_prs(session: requests.Session, submodules: list[dict]) -> None`
   - Initialize ALL submodules' `last_merged_bot_pr` to None first
   - Build query: `repo:{PARENT_OWNER}/{PARENT_REPO} author:{BOT_AUTHOR} is:pr is:merged sort:updated-desc`
   - GET `/search/issues` with params `{"q": query, "per_page": 100, "sort": "updated", "order": "desc"}`
   - On API error: return (all already None)
   - Build `sub_by_name` and `sorted_names` same as above (only status="ok" submodules)
   - Track `matched` set to only keep FIRST (most recent) merged PR per submodule
   - For each PR in response items:
     - Call `match_pr_to_submodule(pr, sorted_names)` 
     - If match found AND name not in `matched`:
       - Parse `pr["pull_request"]["merged_at"]` for merge date (ISO string)
       - Set `sub_by_name[matched_name]["last_merged_bot_pr"] = {"url": pr["html_url"], "merged_at": pr["pull_request"]["merged_at"]}`
       - Add name to `matched`

5. `fetch_latest_repo_commits(session: requests.Session, submodules: list[dict]) -> None`
   - For each submodule:
     - If `sub["status"] != "ok"`: set `sub["latest_repo_commit"] = None`, continue
     - GET `/repos/{sub["owner"]}/{sub["repo"]}/commits/{sub["branch"]}`
     - Set `sub["latest_repo_commit"] = {"url": data["html_url"], "date": data["commit"]["committer"]["date"]}`
     - On error: set `sub["latest_repo_commit"] = None`
     - `time.sleep(0.5)` between submodules

**Style notes:**
- Use `datetime.fromisoformat(date_str.replace("Z", "+00:00"))` for date parsing (established pattern from collector.py)
- Follow staleness.py error handling: per-submodule try/except with None defaults
- Add 0.5s rate-limit courtesy delays between API calls (staleness.py pattern)
